# Reporting a Vulnerability

Reporting a vulnerability is an important way to help improve Keraline OS and keep users safe.

If you discover a security vulnerability in Keraline OS, we encourage you to report it to the developers as soon as possible.

To report a vulnerability:

1. Open an issue in the GitHub Issues tab.
2. Provide a clear description of the vulnerability.
3. Include steps to reproduce the issue if possible.
4. Attach logs, screenshots, or other relevant information.

A member of the Keraline OS team will review the report and respond as soon as possible.

Thank you for helping improve the security and reliability of Keraline OS.
